Dicas de Git e GitHub

Instalando Git e Github

Primeiro, vamos instalar no Windows o Git, nesse link aqui: https://git-scm.com/

PS: O Git também pode ser instalado no Linux, pelo terminal mesmo, procurando pelo pacote "git".

Para o Windows, podemos baixar o Github Desktop, que facilitará por não exigir decorar comandos, baixe aqui: https://desktop.github.com/

No Github Desktop, em options, podemos configurar como por exemplo, o branch padrão em Git ou o editor de texto em Integrations.

Criando Conta no GitHub

Acesse o site do GitHub (https://github.com/)

No GitHub Desktop, coloque seu login e senha. Ele abrirá o navegador para fazer o login, depois voltará ao programa, onde ele mostrará um e-mail temporário para uso. Clique em Finish.

Agora a máquina está pronta para criar repositórios locais e remotos.

Primeiro Repositório Git e GitHub

Vamos criar dois repositórios, um pra HTML e um para Javascript (é recomendado criar repositórios separados para organização).

Clique em Create a New Repository on Your Hard Drive para criar um repositório local (por exemplo, HTML, crie ele na pasta onde está os arquivos, o name pode ser o nome da pasta já com arquivos, caso não queira criar uma nova).

Coloque os arquivos desejados na pasta do programa, depois vá em Commit Main.

PS: Ele criará uma pasta local oculta com o nome .git, onde ficará as diversas versões do site ou programa.

Para publicar remotamente, clique em Publish Repositories (se desejar marcar como privado ou não, pra acessarmos sites ele deverá estar desmarcado).

Ao acessar o site, ele mostrará os seus arquivos lá. Ao abrir o programa, na tela inicial, veremos seus repositórios online, onde podemos cloná-lo pro nosso computador.

Para apagar um repositório, vá no site em Settings do repositório desejado.

Para clonar um repositório de uma URL, basta escolher no GitHub a opção Clone a Repository, vá em URL e coloque a URL ou username Github + repositório.

Para atualizar um repositório existente, basta clonar o repositório, fazer as atualizações no código, fazer um novo commit e clicar em Push Origin.

Hospedando Site no GitHub Pages

No site, vá em Settings e procure a opção GitHub Pages, e escolha um tema, como o minimal, pra habilitar este, e em Sources escolha as opções main e /root. Ele é habilitado para cada repositório independentemente.

Aí você acessará o link https://nomedeusuario.github.io/NomedoRepositorio.

PS: Ele demora um pouco para aparecer online.

Comandos Iniciais do Git

Abra o prompt ou terminal e digite git config --global user.name "Nome de Usuário" pra configurar o nome de usuário, e faça o mesmo com e-mail (trocando o user.name para user.email).

Os arquivos de configuração de usuário ficam no Windows em C:\Users\%username% e no Linux em /home/$USER, o arquivo em questão é o .gitconfig (oculto). Para deslogar, digite git config --global --unset user.name e faça o mesmo trocando o trocando o user.name para user.email.

Crie uma pasta com o comando mkdir (por exemplo mkdir pasta-git) e entre nele com o comando cd. Para iniciar o Git nesse diretório, digite git init . (podendo ignorar o ponto, que significa diretório atual).

Digite git status para ver o status dele. Ele avisará que nada foi adicionado ao commit, mas que existem arquivos. Para adicionar o arquivo, digite git add nomedoarquivo (ou use o coringa * para adicionar tudo). Dê o comando git status de novo para ver o status, agora faça o commit digitando git commit -m "Motivo do Commit".

Podemos digitar git log para ver os dados do commit.

PS: Se o arquivo for alterado, ao rodar git status ele mostrará as alterações do arquivo e nesse caso, daremos o git add e git commit novamente pra adicionar uma atualização. Ele manterá a primeira atualização.

Trabalhar com Repositórios

Como sabemos, repositório é um local de armazenamento onde o projeto e suas alterações são mantidos. Na prática, é uma pasta comum do sistema. Para cada projeto gerenciaado pelo Git, um repositório é configurado.

Para cada projeto, seguimos os seguintes passos:

  • Criar um diretório para armazenar o projeto.
  • Entrar no diretório (opcional).
  • Inicializar o repositório Git.

É criado um subdiretório oculto .git que guardará os dados de alteração e snapshots. A área fora do diretório .git é o diretório de trabalho, onde ficam armazenados os arquivos que iremos manipular.

A staging area é um local para onde os arquivos vão antes de um snapshot ser criado. Somente os arquivos da staging area vão para um snapshot. Selecionamos os arquivos desejado no diretório de trabalho.

Para clonar um repositório, digite git clone https://github.com/NomedoUsuario/NomeDoRepositorio.git.

Para linkar seu projeto para um repositório online, dentro do diretório desejado, digite git init, git remote add origin https://github.com/user/repo.git (menos em atualizações). Digite git add . (não esqueça do ponto) e git commit -m "Motivo do Commit", e depois git push origin master (ele pedirá login e senha).

PS: O login não utiliza a senha do GitHub, e sim a senha do token. Para criar um token, no site do GitHub, vá em Settings, Developer Settings e Personal Access Tokens. Pode ser utilizado main no lugar de master.

Caso seja um novo upload para um repositório já existente e com arquivos, faça a inicialização, a adição e o commit (esqueça do remote add), depois digite os comandos git branch -M master, git pull https://github.com/user/repo.git e git push https://github.com/user/repo.git master -f (cuidado que ele sobrescreverá todos os arquivos, é recomendado clonar um repositório antes de atualizá-lo e upar tudo de uma vez). Caso queira remover um arquivo antes do upload, use o comando git rm arquivo (para diretórios use git rm -rf diretorio).
